## 1. 프로젝트 개요
- **프로젝트명:** Algo-Mate
- **한 줄 소개:** 망각 곡선 이론을 적용한 백준 알고리즘 자동 복습 및 트레이닝 서비스    
- **목표:** 문제 풀이 개수에만 집착하는 기존 방식을 탈피, ***복습***을 통해 사용자의 실제 코딩 테스트 합격률을 높임.    
- **타겟 유저:** 
  - 백준 티어는 높은데 막상 코테에서 떨어지는 취준생
  - 예전에 푼 문제를 자꾸 까먹는 개발자.

## 3. 핵심 기능 명세 

### 메인 대시보드 

사용자가 접속하자마자 **"오늘 무엇을 해야 하는지"** 직관적으로 보여준다.

- **기능:**    
    - **오늘의 할 일 리스트:** [새로 풀 문제]와 [복습할 문제]를 섞어서 카드 형태로 제공.        
    - **티어 연동:** 사용자의 백준 ID를 입력받아 `solved.ac` API를 통해 현재 티어를 가져오고, 다음 티어까지 남은 xp 표시       
    - **스트릭(Streak) 차별화:** 단순 접속이 아니라, '오늘의 할 일'을 완료해야 잔디가 심어짐.
    

###  망각 곡선 기반 복습 시스템 
사용자가 푼 문제를 시스템이 기억했다가 적절한 시점에 다시 제시합니다.

- **로직:**    
    - 사용자가 "풀이 완료" 버튼을 누른 시점 기준 `1일 후` → `3일 후` → `7일 후` → `15일 후` → `30일 후`에 해당 문제를 다시 투하.        
- **UI:**    
    - 복습 문제 카드에는 "🔥 3일 전 풀었던 문제" 태그 표시.        
    - 과거의 내가 남긴 메모(핵심 로직)를 힌트처럼 '보기/숨기기' 할 수 있음.
        


### 마이 리포트

- **기능:**    
    - **유형별 취약점 분석:** (예: 그래프 탐색 정답률 90%, DP 정답률 40% → "DP 위주로 연습하세요!" 메시지)        
    - 차트 라이브러리(`Recharts`)를 활용하여 시각적 만족도 제공.

## 4. 기술적 요구사항 (프론트엔드 중심)

### A. 데이터 관리 (Backend 대체)
- **Supabase** 사용
- **DB 스키마 설계:**

##### profile
| 컬럼          | 데이터 타입    | 설명         |
| ----------- | --------- | ---------- |
| id          | UUID      | 사용자 고유 ID  |
| baekjoon_id | text      | 백준 아이디     |
| goal_streak | int       | 일일 목표 문제 수 |
| created_at  | timestamp | 가입일        |

##### 문제 정보
| 컬럼    | 데이터 타입 | 설명       |
| ----- | ------ | -------- |
| id    | int4   | 백준 문제 번호 |
| title | text   | 문제 제목    |
| level | int4   | 난이도      |
| tags  | jsonb  | 알고리즘 태그  |

##### 오답 노트
| 컬럼명            | 데이터 타입    | 설명        | 비고                                   |
| -------------- | --------- | --------- | ------------------------------------ |
| id             | int8      | 고유 ID     | 자동 증가                                |
| user_id        | uuid      | 사용자 ID    | profile.id                           |
| problem_id     | int4      | 문제 번호     | problem.id                           |
| stage          | int2      | 현재 복습 단계  | 0(오답직후) → 1(1일뒤) → 2(3일뒤)... (기본값 0) |
| next_review_at | timestamp | 다음 복습 날짜  |                                      |
| last_solved_at | timestamp | 마지막 풀이 날짜 |                                      |
| memo           | text      | 사용자 메모    | 마크다운 형식                              |
| is_mastered    | bool      | 졸업 여부     | 더 이상 복습 안함 = 제거 (기본값 false)          |

### B. 외부 API 연동
- **solved.ac API:** 무료로 공개된 비공식 API 활용.    
    
     
### C. 프론트엔드 스택
- **Framework:** React (Vite)    
- **Language:** TypeScript (타입 안정성 확보 및 협업 고려)    
- **State Management:** TanStack Query (React Query)    
- **Styling:** Tailwind CSS
